OQtima Registration Script

Implementation guide for OQtima popup registration script

FOR OQTIMA INTERNAL USE ONLY - This script and implementation guide is restricted to authorized OQtima personnel only

Implementation Guide

Follow these steps to integrate the OQtima registration popup script on your landing page. This new implementation provides more flexibility by allowing you to use any HTML element as a trigger for the registration popup.

Step 1: Add the Script to Your Page

Add the script tag to the <head> section of your HTML. This allows the script to load early and be available throughout your page:

<script src="https://oqtima.com/scripts/registration-popup-script.min.js" data-api-key="your_api_key_here" ></script>
Important: The data-api-key attribute is required for authentication. You must use the API key provided by OQtima. Contact your account manager if you don't have an API key.

Step 2: Add Trigger Elements

Add the data-oqtima-trigger attribute to any HTML element you want to use as a trigger for the registration popup. This can be a button, link, image, or any other clickable element:

<!-- Button trigger example --> <button data-oqtima-trigger data-referral-type="12" data-referral-value="IB08801328J" data-lang="en" > Register Now </button> <!-- Link trigger example --> <a href="#" data-oqtima-trigger data-referral-type="12" data-referral-value="IB08801328J" data-lang="en" > Open Registration Form </a> <!-- Image trigger example --> <img src="path-to-your-banner.jpg" alt="Registration Banner" data-oqtima-trigger data-referral-type="12" data-referral-value="IB08801328J" data-lang="en" />

Live Examples:

Open Registration Form
Supported parameters:
  • data-oqtima-trigger: Required attribute that marks the element as a trigger for the registration popup
  • data-referral-type: Referral type for tracking (optional)
    • 12 - IB Referral Link
    • 14 - Campaign Link
  • data-referral-value: Referral value for tracking (optional)
  • data-lang: Language code (en, fr, br, vn, th, es, it, cn, zh, id, jp, my, ar) (optional, default: "en")
